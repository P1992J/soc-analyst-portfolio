## Executive Summary

A security event log analysis was conducted on a personal Windows workstation to identify any indicators of suspicious activity or compromise. Four key Event IDs were investigated across the Windows Security log. The analysis found **no evidence of malicious activity, unauthorised account creation, or external intrusion**. Both failed login events were individually investigated, fully explained, and confirmed to be benign system behaviour.

---

## Methodology

The following Event IDs were filtered and analysed using Windows Event Viewer:

| Event ID | Description | Relevance |
|----------|-------------|-----------|
| 4624 | Successful account logon | Baseline activity monitoring |
| 4625 | Failed account logon | Detect brute force or unauthorised access |
| 4688 | New process created | Detect unusual or malicious process launches |
| 4720 | New user account created | Detect unauthorised account creation |

### Event ID 4625 — Failed Logons

---

**Failed Logon Event 1 — 23 May 2026 at 18:25:15**

| Field | Value |
|-------|-------|
| Event Record ID | 1199957 |
| Date / Time | 23/05/2026 18:25:15 NZST |
| Subject Account | julit (JULITAPATEL domain) |
| Target Account | Guest (JulitaPatel domain) |
| Logon Type | 3 — Network |
| Failure Reason | Account currently disabled |
| Status Code | 0xC000006E |
| Sub Status | 0xC0000072 |
| Caller Process | C:\Windows\explorer.exe (PID: 0x2068) |
| Source IP | Not available (-) |
| Authentication Package | Negotiate |

**Analysis:**  
User account "julit" on the local machine attempted a network-type logon (Type 3) to the built-in Guest account, which is disabled by default on Windows. This was triggered by Windows Explorer attempting to enumerate network shares or mapped drives and falling back to the Guest account for authentication. Status code 0xC000006E confirms the account exists but is disabled. Sub status 0xC0000072 confirms the specific reason as account disabled.

This is **standard Windows Explorer behaviour** when accessing network resources. The Guest account being disabled is a **security best practice** — the system is correctly configured.

**Verdict:** ✅ Benign — normal Windows Explorer network enumeration behaviour

---

**Failed Logon Event 2 — 24 May 2026 at 20:36:10**

| Field | Value |
|-------|-------|
| Event Record ID | 1202415 |
| Date / Time | 24/05/2026 20:36:10 NZST |
| Subject Account | JULITAPATEL$ (SYSTEM / WORKGROUP) |
| Target Account | Not specified (-) |
| Logon Type | 2 — Interactive |
| Failure Reason | Error occurred during logon |
| Status Code | 0xC000006D |
| Sub Status | 0xC0000380 |
| Caller Process | C:\Windows\System32\svchost.exe (PID: 0xa24) |
| Source IP | 127.0.0.1 (localhost) |
| Authentication Package | Negotiate |

**Analysis:**  
This event was generated by the Windows SYSTEM account via svchost.exe — a core Windows service host process. The source IP of 127.0.0.1 confirms this is an internal localhost request with no external network involvement. Status code 0xC000006D indicates a general authentication failure. Sub status 0xC0000380 specifically indicates a **smartcard or certificate-related failure** — a known and documented Windows behaviour on machines that do not have smartcard readers configured.

This is **completely normal Windows system behaviour** with no security implications.

**Verdict:** ✅ Benign — normal Windows system process behaviour (smartcard policy)

## Recommendations

1. **Enable Process Command Line Auditing** — Configure Windows audit policy to capture full command line arguments in 4688 events for richer forensic detail.
2. **Regular log review** — Schedule monthly security log reviews to establish a normal activity baseline. Deviations are easier to spot once baseline is understood.
3. **Increase log retention** — Configure Security log to retain a minimum of 90 days of history to support meaningful forensic analysis.
4. **Consider SIEM integration** — Forward Windows Security logs to Splunk (free tier) for real-time alerting on patterns such as repeated 4625 events or unusual 4688 process launches.
5. **Guest account** — Confirm Guest account remains disabled. This is already correctly configured on this system.
